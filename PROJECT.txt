================================================================
  OurStone — Project Description
================================================================

About the project
-----------------
OurStone is a Slovak online store — a full-featured e-commerce web
application with a customer-facing interface and an admin panel.

URL: https://ourstone.fun

================================================================

Technologies
------------
Framework:        Next.js 16 (App Router, Turbopack)
Language:         TypeScript
Styling:          Tailwind CSS v4
Database:         Neon PostgreSQL (serverless)
ORM:              Prisma v7 (with @prisma/adapter-pg)
Authentication:   NextAuth.js v5 (Credentials provider, JWT sessions)
Payments:         Stripe (CardElement, confirmCardPayment)
Charts:           Recharts (AreaChart, BarChart)
Hosting:          Vercel
Email:            Resend (custom domain ourstone.fun)
Icons:            Lucide React
Font:             Geist (Google Fonts)

Database (Neon PostgreSQL + Prisma)
------------------------------------
15 models in Prisma schema (prisma/schema.prisma):

  User                 User (email, bcrypt hash, name, role, blocked)
  Address              Customer shipping addresses
  Product              Products (name, price, image, category, stock, discount,
                       variants, nameEn, descriptionEn)
  Review               Reviews (customer, product, rating 1–5, text,
                       status: pending/approved/rejected)
  Order                Orders (customer, address, total, status, date, notes)
  OrderItem            Order items (name, price, quantity)
  Favorite             User's favorite products
  PasswordResetToken   Password reset tokens (1h expiration)
  LoginAttempt         Brute-force tracking (email+ip composite key, lockedUntil)
  Coupon               Discount coupons (%, fixed, shipping; usedCount, maxUses, expiresAt)
  Setting              Key-value store for shop settings
  Page                 CMS pages (slug, title, content)
  AdminSession         Admin login records (ip, userAgent, loginAt, logoutAt)
  AdminLog             Admin action audit log (userId, email, action, entity, detail)
  PageView             Website visit records (path, referrer, ip, userAgent)

Prisma client: src/lib/prisma.ts (singleton with PrismaPg adapter)
Schema push: npx prisma db push
Seed script: prisma/seed.ts (10 products + super_admin account)

Authentication (NextAuth.js v5)
--------------------------------
Configuration: src/lib/auth.ts
Provider:      Credentials (email + password, bcrypt comparison)
Session:       JWT strategy (role in token)
Route handler: src/app/api/auth/[...nextauth]/route.ts
TypeScript:    src/types/next-auth.d.ts (roles in Session and JWT)

Roles: customer, editor, support, admin, super_admin

Role permissions (src/lib/permissions.ts):
  super_admin  all features (including analytics and user management)
  admin        all features (including analytics)
  editor       dashboard, products
  support      dashboard, orders, customers, reviews

Two separate React contexts:
  - UserAuthContext    for customers (useSession + signIn/signOut)
  - AdminAuthContext   for admins (useSession + role check + can() helper)

Brute-force protection (src/lib/auth.ts + LoginAttempt table):
  - Max 5 failed attempts per email+IP combination
  - After 5 failures: account locked for 15 minutes
  - On successful login: attempt records deleted

Forgot password (custom token-based flow):
  1. Customer enters email at /forgot-password
  2. API creates token + sends email via Resend
  3. Clicks link -> /reset-password?token=xxx
  4. API verifies token + sets new bcrypt password
  5. Redirected to /password-changed

Payments — Stripe
-----------------
Mode:          Test mode (acct_1S4j3ZAFVtb5tsxW)
Element:       CardElement (hidePostalCode: true)
Flow:          2-step checkout:
               Step 1: Contact + shipping details + coupon code
               Step 2: Stripe card payment

Payment flow:
  1. User fills form, optionally applies coupon
  2. Clicks "Continue to payment" -> creates PaymentIntent
  3. Enters card -> confirmCardPayment()
  4. On success: saves order to DB, increments coupon usedCount, sends email

Test card:     4242 4242 4242 4242 | exp: 12/28 | CVV: 123
Test modal:    Shown on checkout entry, language-aware (SK/EN)
Countdown:     20s countdown + circular progress after order confirmation

Stripe Webhooks:
  Endpoint:    https://ourstone.fun/api/webhooks/stripe
  Events:      payment_intent.succeeded -> order status "zaplatená"
               payment_intent.payment_failed -> order status "zrušená"
  Validation:  stripe.webhooks.constructEvent() with STRIPE_WEBHOOK_SECRET

Coupons
-------

Email — Resend
--------------
Domain:    ourstone.fun (verified DNS records on Websupport)
From:      noreply@ourstone.fun

DNS records (added on Websupport):
  - DKIM TXT   resend._domainkey.ourstone.fun
  - MX         -> send.smtp.resend.com
  - SPF TXT    v=spf1 include:spf.resend.com -all

Emails sent:
  1. Contact form -> notification to owner
  2. Contact form -> auto-reply to customer
  3. Order confirmation -> customer (bilingual SK/EN template)
  4. New order -> notification to owner
  5. Password reset -> customer (link with token)
  6. New review -> notification to admin (link to admin panel)
  7. Review approved -> customer
  8. Review rejected -> customer

Review system
-------------
Flow: logged-in customer -> submit -> email to admin -> approve/reject -> email to customer

Customer:
  - Submits review on product page (1–5 stars + text comment)
  - Status "pending" — not publicly visible until admin approves
  - Receives email after approval or rejection

Admin:
  - /admin/reviews — all reviews with filter (pending / approved / rejected)
  - Badge showing pending review count in sidebar
  - Approve / Reject buttons -> automatically sends email to customer

API:
  GET  /api/reviews?productId=X    approved reviews for a product (public)
  POST /api/reviews                submit new review (logged-in customer) + email to admin
  GET  /api/admin/reviews          all reviews (admin)
  PATCH /api/admin/reviews         approve/reject + email to customer + audit log

Admin audit log
---------------
Model: AdminLog (helper function logAdminAction() in src/lib/adminLog.ts)

Tracked actions:
  - Products:    create / update / delete
  - Orders:      create / update (status) / note
  - Customers:   block / unblock
  - Coupons:     create / update / toggle / delete
  - Reviews:     approve / reject
  - Admins:      create (new account)

Display:
  - /admin/settings -> Activity tab (super_admin only)
  - Sub-tab "Audit log" — table: time, admin email, action (colored badge), entity, detail
  - Sub-tab "Login history" — sessions table (name, IP, browser, duration)
  - Super admin sees login history for all admins

API:
  GET /api/admin/logs      last 200 log entries (super_admin only)
  GET /api/admin/sessions  sessions (super_admin sees all, others see own only)

Custom analytics
----------------
Self-hosted tracking without external services (e.g. Google Analytics).

What is recorded (PageView model):
  - Page URL (path)
  - Where visitor came from (referrer)
  - IP address (for unique visitor counting)
  - User agent (for device detection)
  - Visit timestamp

Filtering:
  - Admin pages (/admin/*) are not tracked
  - Bots (crawler, spider, Facebook, Twitter, Slack...) are filtered out

Component: src/components/AnalyticsTracker.tsx
  - "use client", usePathname() hook — captures every URL change
  - Calls POST /api/analytics on each route change

Admin page /admin/analytics:
  - 4 stat cards: Today / This week / Last X days / Unique visitors
  - Daily bar chart (hover = count, pure CSS — no chart library)
  - Top 10 most visited pages with progress bars
  - Top 8 traffic sources (referrer domain)
  - Device breakdown: Desktop / Mobile / Tablet (%)
  - Range selector: 7 / 14 / 30 / 90 days

API:
  POST /api/analytics               record visit (public, bot-filtered)
  GET  /api/admin/analytics?range=X aggregated stats (admin/super_admin only)

Internationalization (SK / EN)
-------------------------------
Language toggle in header (Globe icon + 2-letter code).
Language stored in localStorage (key: shopsk_lang).
Default language: Slovak (sk).

Implementation:
  src/context/LanguageContext.tsx
    - hook useLanguage() -> { lang, setLang, t, tCat }
    - t(key)     translates key by active language
    - tCat(name) translates category names (SK -> EN)
  src/lib/translations.ts
    - Two dictionaries: sk{} and en{} with identical keys

Product translations:
  Product.nameEn / Product.descriptionEn — optional EN fields in DB
  ProductCard + product detail show nameEn/descriptionEn when lang=en
  Falls back to Slovak if EN field is empty
  Admin form includes "Name (EN)" and "Description (EN)" fields

Security
--------
- Authorization at every API route level (auth() from NextAuth)
- Customers can only access their own data (orders, addresses, favorites)
- Admin API routes verify role and return 401 for unauthorized access
- Role-based UI: sidebar shows only permitted items per role
- super_admin has exclusive access: user management, audit log, activity tab
- Blocked customers cannot log in (checked in auth.ts)
- Passwords hashed via bcryptjs (12 rounds)
- JWT session — no database sessions
- Brute-force protection: 5 attempts -> 15 min lock (email+IP composite key)
- Stripe webhook signature validation (constructEvent + STRIPE_WEBHOOK_SECRET)
- GDPR cookie consent banner (localStorage, SK/EN)
- Coupon validation: active flag, expiry date, maxUses limit, minOrder threshold
- Analytics: bots are filtered out, admin pages are not tracked
- Env variables not exposed to client
