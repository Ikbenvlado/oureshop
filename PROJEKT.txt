================================================================
  OurStone — Popis projektu
================================================================

Čo je to za projekt
--------------------
OurStone je slovenský online obchod — e-commerce webová aplikácia
s plnohodnotným zákazníckym rozhraním aj admin panelom.

URL: https://ourstone.fun

================================================================

Technológie
-----------
Framework:        Next.js 16 (App Router, Turbopack)
Jazyk:            TypeScript
Štýlovanie:       Tailwind CSS v4
Databáza:         Neon PostgreSQL (serverless)
ORM:              Prisma v7 (s @prisma/adapter-pg)
Autentifikácia:   NextAuth.js v5 (Credentials provider, JWT sessions)
Platby:           Stripe (CardElement, confirmCardPayment)
Grafy:            Recharts (AreaChart, BarChart)
Hosting:          Vercel
Email:            Resend (vlastná doména ourstone.fun)
Ikony:            Lucide React
Font:             Geist (Google Fonts)

Databáza (Neon PostgreSQL + Prisma)
------------------------------------
15 modelov v Prisma schéme (prisma/schema.prisma):

  User                 Používateľ (email, bcrypt hash, meno, rola, blocked)
  Address              Doručovacie adresy zákazníkov
  Product              Produkty (názov, cena, obrázok, kategória, sklad, zľava,
                       varianty, nameEn, descriptionEn)
  Review               Recenzie (zákazník, produkt, hodnotenie 1–5, text,
                       stav pending/approved/rejected)
  Order                Objednávky (zákazník, adresa, suma, stav, dátum, poznámky)
  OrderItem            Položky objednávky (názov, cena, množstvo)
  Favorite             Obľúbené produkty používateľa
  PasswordResetToken   Tokeny na obnovenie hesla (1h expirácia)
  LoginAttempt         Sledovanie brute-force pokusov (email+ip kľúč, lockedUntil)
  Coupon               Zľavové kupóny (%, fixné, doprava; usedCount, maxUses, expiresAt)
  Setting              Key-value úložisko nastavení obchodu
  Page                 CMS stránky (slug, title, content)
  AdminSession         Záznamy prihlásení adminov (ip, userAgent, loginAt, logoutAt)
  AdminLog             Audit log akcií adminov (userId, email, action, entity, detail)
  PageView             Záznamy návštev webu (path, referrer, ip, userAgent)

Prisma klient: src/lib/prisma.ts (singleton s PrismaPg adaptérom)
Schéma push: npx prisma db push
Seed skript: prisma/seed.ts (10 produktov + super_admin účet)

Autentifikácia (NextAuth.js v5)
--------------------------------
Konfigurácia:  src/lib/auth.ts
Provider:      Credentials (email + heslo, bcrypt porovnanie)
Session:       JWT stratégia (rola v tokene)
Route handler: src/app/api/auth/[...nextauth]/route.ts
TypeScript:    src/types/next-auth.d.ts (roly v Session a JWT)

Roly: customer, editor, support, admin, super_admin

Oprávnenia podľa roly (src/lib/permissions.ts):
  super_admin  všetko (vrátane analytiky a správy používateľov)
  admin        všetko (vrátane analytiky)
  editor       dashboard, produkty
  support      dashboard, objednávky, zákazníci, recenzie

Dva samostatné React kontexty:
  - UserAuthContext    pre zákazníkov (useSession + signIn/signOut)
  - AdminAuthContext   pre adminov (useSession + kontrola role + can() helper)

Ochrana proti brute-force (src/lib/auth.ts + LoginAttempt tabuľka):
  - Max 5 neúspešných pokusov na kombináciu email+IP
  - Po 5 zlyhaniach: zámok na 15 minút (lockedUntil)
  - Po úspešnom prihlásení: záznamy sa vymažú

Zabudnuté heslo (custom token-based flow):
  1. Zákazník zadá email na /forgot-password
  2. API vytvorí token + pošle email cez Resend
  3. Klikne na link → /reset-password?token=xxx
  4. API overí token + nastaví nové bcrypt heslo
  5. Presmerovaný na /password-changed

Platby — Stripe
---------------
Režim:         Test mode (acct_1S4j3ZAFVtb5tsxW)
Element:       CardElement (hidePostalCode: true)
Flow:          2-krokový checkout:
               Krok 1: Kontaktné + doručovacie údaje + kód kupóna
               Krok 2: Platba kartou cez Stripe

Priebeh platby:
  1. Zákazník vyplní formulár, prípadne zadá kupón
  2. Klikne "Pokračovať na platbu" → vytvorí sa PaymentIntent
  3. Zadá kartu → confirmCardPayment()
  4. Pri úspechu: uloží objednávku do DB, inkrementuje usedCount kupónu, pošle email

Testovacia karta:  4242 4242 4242 4242 | exp: 12/28 | CVV: 123
Test modal:        Zobrazí sa pri vstupe na checkout, jazykovo citlivý (SK/EN)
Odpočítavanie:     20s countdown + kruhový progres po potvrdení objednávky

Stripe Webhooky:
  Endpoint:    https://ourstone.fun/api/webhooks/stripe
  Eventy:      payment_intent.succeeded → stav objednávky "zaplatená"
               payment_intent.payment_failed → stav objednávky "zrušená"
  Validácia:   stripe.webhooks.constructEvent() s STRIPE_WEBHOOK_SECRET

Kupóny

Email — Resend
--------------
Doména:    ourstone.fun (overená DNS záznamy na Websupport)
Od:        noreply@ourstone.fun

DNS záznamy (pridané na Websupport):
  - DKIM TXT   resend._domainkey.ourstone.fun
  - MX         → send.smtp.resend.com
  - SPF TXT    v=spf1 include:spf.resend.com -all

Odosielané emaily:
  1. Kontaktný formulár → notifikácia vlastníkovi
  2. Kontaktný formulár → auto-reply zákazníkovi
  3. Potvrdenie objednávky → zákazníkovi (dvojjazyčná SK/EN šablóna)
  4. Nová objednávka → notifikácia vlastníkovi
  5. Obnovenie hesla → zákazníkovi (link s tokenom)
  6. Nová recenzia → notifikácia adminovi (link do admin panelu)
  7. Schválenie recenzie → zákazníkovi
  8. Zamietnutie recenzie → zákazníkovi

Systém recenzií
---------------
Tok: prihlásený zákazník → odoslanie → email adminovi → schválenie/zamietnutie → email zákazníkovi

Zákazník:
  - Odoslie recenziu na stránke produktu (hviezdičky 1–5 + text)
  - Stav "pending" — nezobrazuje sa verejne kým admin neschváli
  - Dostane email po schválení alebo zamietnutí

Admin:
  - /admin/reviews — prehľad všetkých recenzií s filtrom (čakajúce / schválené / zamietnuté)
  - Badge s počtom čakajúcich recenzií v sidebari
  - Tlačidlá Schváliť / Zamietnuť → automaticky sa odošle email zákazníkovi

API:
  GET  /api/reviews?productId=X    schválené recenzie produktu (verejné)
  POST /api/reviews                odoslanie novej recenzie (prihlásený zákazník) + email adminovi
  GET  /api/admin/reviews          všetky recenzie (admin)
  PATCH /api/admin/reviews         schválenie/zamietnutie + email zákazníkovi + audit log

Audit log adminov
-----------------
Model: AdminLog (src/lib/adminLog.ts — pomocná funkcia logAdminAction())

Zaznamenávané akcie:
  - Produkty:    create / update / delete
  - Objednávky:  create / update (stav) / note
  - Zákazníci:   block / unblock
  - Kupóny:      create / update / toggle / delete
  - Recenzie:    approve / reject
  - Admini:      create (nový účet)

Zobrazenie:
  - /admin/settings → záložka Aktivita (len super_admin)
  - Podzáložka "Audit log" — tabuľka: čas, admin email, akcia (farebný badge), entita, detail
  - Podzáložka "Prihlásenia" — história sessions (meno, IP, zariadenie, trvanie)
  - Super admin vidí prihlásenia všetkých adminov

API:
  GET /api/admin/logs      posledných 200 záznamov (len super_admin)
  GET /api/admin/sessions  sessions (super_admin vidí všetky, ostatní len vlastné)

Vlastná analytika
-----------------
Sledovanie bez externých služieb (napr. Google Analytics).

Čo sa zaznamenáva (model PageView):
  - URL stránky (path)
  - Odkiaľ návštevník prišiel (referrer)
  - IP adresa (pre unikátnych návštevníkov)
  - User agent (pre detekciu zariadenia)
  - Čas návštevy

Filtrovanie:
  - Admin stránky (/admin/*) sa nezaznamenávajú
  - Boti (crawler, spider, Facebook, Twitter, Slack...) sa filtrujú

Komponent: src/components/AnalyticsTracker.tsx
  - "use client", hook usePathname() — zachytáva každú zmenu URL
  - Volá POST /api/analytics pri každej zmene stránky

Admin stránka /admin/analytics:
  - 4 stat karty: Dnes / Tento týždeň / Za X dní / Unikátni návštevníci
  - Stĺpcový graf po dňoch (hover = počet, čistý CSS bez knižníc)
  - Top 10 najnavštevovanejších stránok s progress barmi
  - Top 8 zdrojov návštevnosti (doména referrera)
  - Rozdelenie zariadení: Desktop / Mobil / Tablet (%)
  - Prepínač rozsahu: 7 / 14 / 30 / 90 dní

API:
  POST /api/analytics               zaznamenanie návštevy (verejné, bot filter)
  GET  /api/admin/analytics?range=X agregované štatistiky (len admin/super_admin)

Viacjazyčnosť (SK / EN)
-----------------------
Prepínač jazyka je v headeri (Globe ikona + 2-písmenový kód).
Jazyk sa ukladá do localStorage (kľúč: shopsk_lang).
Predvolený jazyk: slovenčina (sk).

Implementácia:
  src/context/LanguageContext.tsx
    - hook useLanguage() → { lang, setLang, t, tCat }
    - t(key)     prekladá kľúč podľa aktívneho jazyka
    - tCat(name) prekladá názvy kategórií (SK → EN)
  src/lib/translations.ts
    - Dve slovníky: sk{} a en{} s rovnakými kľúčmi

Preklady produktov:
  Product.nameEn / Product.descriptionEn — voliteľné EN polia v DB
  ProductCard + product detail zobrazujú nameEn/descriptionEn keď lang=en
  Fallback na SK ak EN pole nie je vyplnené
  Admin formulár obsahuje polia "Názov (EN)" a "Popis (EN)"

Bezpečnosť
----------
- Autorizácia na úrovni každej API route (auth() z NextAuth)
- Zákazník vidí len vlastné dáta (objednávky, adresy, obľúbené)
- Admin API routes overujú rolu a vracajú 401 pri neautorizovanom prístupe
- UI sidebaru zobrazuje len povolené položky podľa role
- super_admin má exkluzívny prístup: správa používateľov, audit log, aktivita
- Blokovaní zákazníci sa nemôžu prihlásiť
- Heslá hashované cez bcryptjs (12 rounds)
- JWT session — žiadne databázové sessions
- Brute-force ochrana: 5 pokusov → 15 min zámok (email+IP kľúč)
- Stripe webhook overenie podpisu (constructEvent + STRIPE_WEBHOOK_SECRET)
- GDPR cookie consent banner (localStorage, SK/EN)
- Kupóny: overenie active flag, expirácie, maxUses, minOrder
- Analytika: boti sa filtrujú, admin stránky sa nezaznamenávajú
